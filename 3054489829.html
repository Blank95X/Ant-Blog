编译原理 第一章 概论

发表于 2025-04-07 分类于 编译原理

概论

什么是编译程序

程序语言

对象 内容
低级语言 机器语言、汇编语言等面向机器的语言
高级语言 C、C++、Java等接近自然语言的语言

使用翻译程序从高级翻译到机器语言。

翻译程序

翻译程序可分为：

对象 内容
解释程序 边解释边执行
汇编程序 汇编语言->机器语言
编译程序 生成完全独立于源程序、经优化的目标语言代码；屏蔽硬件细节 
graph TD
    HL[高级语言]
    AL[汇编语言]
    ML[机器语言]
    compiling{编译}
    assembling{汇编}

    HL --> compiling
    compiling --> AL
    AL --> assembling
    assembling --> ML
    compiling --> ML

高级语言程序执行过程

编译阶段：完成源程序到目标程序的翻译。

运行阶段：必须有系统库支持。

graph TD
    A[源程序]
    B{编译程序}
    H{汇编程序}
    C[机器语言目标程序]
    D(系统库)
    E(初始数据)
    F[结果]
    G[汇编语言目标程序]

    A --> B
    B --> G
    subgraph 若汇编语言为目标语言
        G -.-> H
    end
    H --> C
    D --> C
    E --> C
    C --> F

编译系统

将编译程序、链接程序、系统库、源程序编辑程序等软件合称为编译系统。

编译程序还可以进一步分类：

分类 功能
诊断编译程序 帮助程序开发和调试
优化编译程序 提高目标代码效率
交叉编译程序 在一种机器上编译另一种机器的程序
可变目标编译程序 生成多种目标代码

编译的过程

词法分析

对构成源程序的字符串进行分解，根据源程序的程序语言的词法规则，识别词法单元（单词符号，也叫符号）。

词法规则

单词符号的形成规则，用正规式和有限自动机描述。

例子

 
float r,h,s;
s=2*3.1416*r*(h+r);

识别出如下单词符号：

单词符号 类型
float 关键字
r 标识符
h 标识符
s 标识符
= 运算符
2 常数
3.1416 常数
* 运算符
( 界符
) 界符
; 界符
* 运算符

语法分析

在词法分析的基础上，根据语法规则识别出语法单元，并进行语法检查。

语法规则

规定单词符号如何形成语法单元，用上下文无关法描述。

<赋值语句> -> <变量> = <表达式>

语义分析和生成中间代码

对每个语法单元进行静态语义审查，分析其含义，然后用中间语言来描述这种语义。

语义规则

用属性文法来描述语义规则。

代码优化

对中间代码进行等价的变换或改造，提高时间和空间效率

生成目标代码

将中间代码变为目标代码。

表格管理和错误处理

编译的各个阶段都涉及操作相关表格的信息，必须要有程序来管理表格。

编译程序的结构

    error[错误处理程序]
    table[表格管理程序]

    table <-.->  B 
    table <-.->  C 
    table <-.->  D 
    table <-.->  E 
    table <-.->  F 

    B <-.-> error
    C <-.-> error
    D <-.-> error
    E <-.-> error
    F <-.-> error

        A[源程序（字符串）] --> B[词法分析程序]
    subgraph 编译过程
        B --> C[语法分析程序]
        C --> D[语义分析和中间代码生成程序]
        D --> E[代码优化程序]
        E --> F[目标代码生成程序]
    end
        F --> G[目标程序]

设计编译程序的逻辑：

  1. 分析源语言和目标语言
  2. 设计编译算法
  3. 选择编译程序的语言
  4. 调试
  5. 提交文档

编译程序的前后端

前端：与源语言有关、与目标语言无关的部分。

前端包括：词法分析、语法分析、语义分析、中间代码生成、符号表管理、机器无关代码优化。

后端：与目标语言有关的部分。

后端包括：机器有关代码优化、目标代码生成、错误处理、符号表操作。

扫描

对源程序或中间代码进行扫描并加工，产生新的中间代码或目标程序。

采用不同的扫描遍数和分遍方式，会造成编译程序在具体结构上的差别。

分遍的主要好处：

  • 减少内存占用；
  • 优化更充分，目标程序的质量更高；
  • 将编译程序的前端和后端分开，便于移植。

分遍的主要缺点：增加重复工作，降低编译效率。

是否分遍和分遍次数，要根据具体情况来决定（计算机内存大小、程序语言的简繁、目标程序质量高低等）。

编译程序的生成

任何一个编译程序都涉及到3个语言：源语言、目标语言、编写编译器的语言。

由于编写第一个编译器时不存在任何其他编译器，因而该编译程序只能用机器语言编写。

自展

使用目标语言自身编写编译程序（自己编译自己）。

对于源语言A、编写语言C：

  • 首先使用C编写A的一个核心子集 A1A_1A1​ 的编译器。
  • 使用源语言A编写一个更为完整的子集 A2A_2A2​ 的编译器（ A1∈A2A_1 \in A_2A1​∈A2​ ），使用上一步得到的编译器进行编译。
  • 重复上一步，直到编译器的功能达到要求。
